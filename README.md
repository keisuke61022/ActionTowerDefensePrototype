# Action Tower Defense Prototype

## Unityでの確認手順
1. Unity 6.4でこのプロジェクトを開く
2. 任意のSceneを開く（この実装は再生時に自動でゲームオブジェクト生成）
3. Playボタンを押す
4. そのままCPU戦プロトタイプが開始

## 操作方法
- 移動: `WASD` または `矢印キー` または 左下バーチャルスティック
- 射撃: `Space` / 左クリック / 右下 `射撃`
- ユニット1設置(砲台): `E` / 右下 `砲台`
- 未実装ボタン（`近接` / `兵士` / `範囲` / `回復`）を押すと「未実装」表示

## 今回の修正内容（Input/UI/見た目）
- 入力処理を `UnityEngine.Input` 依存から `Unity Input System`（`Keyboard.current` / `Mouse.current`）へ移行
- EventSystem を `StandaloneInputModule` から `InputSystemUIInputModule` へ変更
- 戦場を覆って見えづらかった背景表示を調整し、URP Unlitマテリアルでプレイヤー/敵/拠点の色が常に判別できるように修正
- 9:16縦画面向けにUIの余白と高さを調整し、操作ボタンとコストゲージが重ならないように修正
- 再生直後に即敗北しないこと、約1.5秒後から敵が湧いて自拠点へ進軍する導線を維持

## 実装済み内容
- ゲーム開始時に `GameManager` が戦場・拠点・プレイヤー・UIを自動生成
- 敵5ウェーブを自動生成
- プレイヤー射撃と砲台の自動攻撃
- コストシステム（初期5、最大10、設置3、撃破+1）
- 勝利/敗北表示（敵拠点破壊 or 5ウェーブ生存 / 自拠点破壊）

## 今回の追加修正（操作エリア分離/UI改善）
- プレイヤー移動を戦闘エリア矩形でClampし、下部の操作UI・コストゲージ領域へ侵入しないように調整
- 左下バーチャルスティックに丸いノブを追加（ドラッグ追従、可動範囲制限、指を離すと中央復帰）
- 攻撃/ユニットボタンを日本語化（射撃、近接、砲台、兵士、範囲、回復）
- 砲台ボタンを2行表示（砲台 / 3コスト）に変更し、コスト不足時は設置不可＋「コスト不足」表示
- 未実装ボタンをグレー表示に統一し、押下時に「未実装」表示
- 右下ボタン群を2列×3段で再配置し、コストゲージ・スティックエリアと重ならないよう調整
